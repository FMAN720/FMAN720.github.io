学习点1：输入文件流对象，在函数loadTasks()中

学习原因是因为在代码中发现


代码在项目中的详细意思是：
std::ifstream 是 C++ 标准库中的类，用于创建输入文件流对象，从文件读取数据。
file(FILE_NAME) 创建一个名为 file 的 std::ifstream 对象，并尝试以只读模式打开名为 FILE_NAME（常量字符串 “tasks.json”）的文件。
如果文件存在且可访问，file 对象可用于后续读取操作；如果失败，file.is_open() 将返回 false（下一行代码会检查并处理）。
在 loadTasks 函数上下文中，这行代码用于打开任务数据文件，为后续从 JSON 文件加载任务列表做准备。

形象比喻

  1. 核心比喻：数据水管
    想象一下：
    你的硬盘上的 tasks.json 文件：是一个装满水的水库（水就是文件里的文本数据）。
    你的 C++ 运行程序（内存）：是一个空水桶，现在急需把文件里的数据装进来处理。
    水库里的水不可能瞬间“瞬移”到水桶里。你需要买一根水管，一头插进水库，一头放在水桶里，让水一点点流（Stream）过来。
    在 C++ 里：
    流（Stream）：指的就是数据像流水一样，一滴一滴（一个字节一个字节）地传输。
    输入（Input）：站在程序的视角，数据从外面（硬盘）进到程序里，叫输入。
    对象（Object）：就是你在代码里造出来的这根“水管”的实体。
    所以，输入文件流对象，就是C++专门用来从硬盘文件里“往程序里抽水（读数据）”的那根管子。它的英文真名叫做 ifstream (Input File Stream)。
    个人理解：
    创建了一个名字叫做：file的对象，可以把这个对象理解为一个水管，这个水管是插入到FILE_NAME这个里面的（也就是tasks.json），

    学习点2：既然有输入文件流，就有输出文件流

    同理，这个就是在task.json文件中加数据（往硬盘里的文件里喷数据。），类似的东西就是cout<<”hello world”;，这不过这个是在屏幕上喷东西，
    学习代码 一些小细节 既然输出流有一个缓冲区，那么输入流就肯定也有，接下来是具体操作流程
  2. 输入流的工作流程（以你的 loadTasks 为例）
    当你执行 buffer << file.rdbuf(); 或者读取文件内容时，幕后发生了这些事：
    预读（Read-ahead）：
    当你让 file 去读 1 个字节时，操作系统其实并不只读这 1 个字节。它会“自作聪明”地从硬盘里直接搬一大块数据（比如 4KB 或 8KB）出来，塞进内存里的输入缓冲区。
    内存取货：
    接下来的读取操作（比如你想找下一个引号、下一个逗号），程序其实根本没去碰硬盘，而是直接在内存里的缓冲区里“翻找”。这速度快得飞起！
    自动补货：
    当缓冲区里的数据被你读光了，而文件还没读完，系统会自动再去硬盘里搬下一块数据进来填满“储水罐”。
    进阶
    如果你想强制“清空”缓冲区？
    对于输出流：有时候你写了数据，但还没攒够一桶，它没写进硬盘。如果你急着要看，可以用 file << std::endl; 或者 file.flush();。这就像是强行踩一下马桶冲水键，不管桶里有多少，全部冲进硬盘。
    对于输入流：通常不需要手动干预，它会自动预读。
